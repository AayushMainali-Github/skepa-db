# skepa-db v1.1.0

## Summary

- `v1.1.0` focuses on database behavior improvements inside the existing `v1` API and storage-format line.
- Adds not-equal filtering, column defaults, schema introspection, richer execution stats, and clearer stable error codes.

## Query And Execution

- Added `!=` and `neq` predicates for `where` filters.
- `select` results now include execution stats for:
  - `rows_scanned`
  - `index_used`
- Stats are populated for full scans, primary-key lookups, unique lookups, and secondary-index lookups.

## Schema

- Added column defaults with `default <literal>` in `create table`.
- Defaults are validated against the column datatype at table creation time.
- Defaults apply to omitted trailing insert values.
- Explicit `null` remains `null` and is not replaced by the default.
- Added `describe <table>` for structured schema introspection.

## API Compatibility

- API version remains `v1`.
- Added stable error codes:
  - `INVALID_DEFAULT`
  - `UNKNOWN_TABLE`
  - `UNKNOWN_COLUMN`
  - `DUPLICATE_INDEX`
  - `UNSUPPORTED_SYNTAX`
- Existing response envelopes remain unchanged.

## Storage Compatibility

- Storage format version remains `1`.
- Column defaults are additive catalog metadata.
- Existing v1.0 databases without default metadata remain loadable.
- Databases with newer unsupported storage format versions are still rejected explicitly.

## Recovery And Transactions

- Added coverage for `!=` update/delete behavior through rollback and commit.
- Added WAL recovery coverage for inserts that use defaults.
- Added restart coverage for `describe` and execution stats.

## Upgrade Notes

- Direct open is expected from `v1.0.x` when `storage_format_version` remains `1`.
- Back up or export databases before upgrading.
- If direct open fails because of a future storage-format boundary, use export/import as the fallback migration path.

## Operator Action

- Review `docs/sql-dialect.md` for new syntax and default semantics.
- Review `docs/api.md` for the expanded error-code list.
- No config migration is required for `v1.1.0`.
