# skepa-db v1.0.0

## Summary

- First stable `v1` release of `skepa-db` as a single-node SQL-style database product.
- Includes the embeddable core engine, HTTP server runtime, remote-capable CLI, session-scoped transactions, multi-database operations, admin endpoints, compatibility metadata, and documented upgrade guidance.

## HTTP API Compatibility

- API version: `v1`
- Compatibility status: documented `v1` HTTP routes, request/response meanings, and stable error codes are intended to remain stable within the `v1` line.

## Storage Compatibility

- Storage format version: `1`
- Direct open/upgrade status: direct open is expected when `storage_format_version` remains `1`.
- Databases with a newer unsupported storage format are rejected explicitly on open.

## Upgrade Notes

- Check `GET /version` and `GET /config` before upgrading.
- Back up the database directory or export named databases before changing binaries.
- If `storage_format_version` remains unchanged, direct open is the expected upgrade path.
- If direct open is blocked by a storage-format boundary, use named database export/import as the supported recovery path.

## Operator Action

- Recommended:
  - place `skepa_db_server` behind TLS termination
  - enable bearer auth for non-local deployments
  - review `docs/server-operations.md`
  - review `docs/upgrades.md` before rollout
